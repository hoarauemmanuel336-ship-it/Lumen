1. LE PROJET
────────────────────────────────────────────────────────────────────
Lumen Veritatis (lumenveritatis.net) : site de théologie catholique bilingue
(français + anglais), destiné aux débutants comme à ceux qui veulent approfondir.
Emmanuel (francophone réunionnais, catholique, non-codeur, Windows + iOS, UTC+4)
écrit lui-même tout le contenu théologique ; l'assistant gère tout le code,
les intégrations, les validations et les exports de fichiers.

Théologie : doctrine catholique, niveau élevé mais accessible. L'assistant donne
directement la lecture qu'il juge la plus juste (patristique, Tradition, cohérence
interne), sans énumérer d'autres interprétations.

────────────────────────────────────────────────────────────────────
2. ARCHITECTURE DES FICHIERS (5 fichiers de site + ce fichier de reprise)
────────────────────────────────────────────────────────────────────
Le site est STATIQUE, hébergé sur Netlify, généré par un script Node.

• build-bi.js — LE GÉNÉRATEUR. Lit les articles depuis index__48_.html, contient
  les chaînes d'interface françaises (menu, pied de page, page « à propos »…),
  le login du site, et produit le site multipage dans le dossier « site/ ».
  Commande de build : node build-bi.js
  Sortie attendue : « Paires de pages : 63 → 126 pages (FR + EN) … »
  Il copie memoriser.html dans site/ à la fin du build.
  La nav du site contient Accueil, Bibliothèque, Mémoriser, À propos, langue,
  nouveautés, recherche et compte.

• index__48_.html — SOURCE FRANÇAISE (le SPA). Fait AUTORITÉ pour tout le
  contenu FR : articles, pied de page, page « à propos ». build-bi.js lit ses
  articles. C'est un fichier unique (SPA, routage par hash, articles = objets JS).

• data-en.js — DONNÉES ANGLAISES. Fait AUTORITÉ pour tout le contenu EN
  (SLUGS, THEMES_EN, UI_EN, ARTICLES_EN, NOUVEAUTES_EN). Requis par build-bi.js.

• memoriser.html — APP DE MÉMORISATION des versets (page autonome, bilingue),
  servie à lumenveritatis.net/memoriser.html. Fichier autonome avec connexion
  Firebase, répétition espacée, catégories, progression, corbeille, restauration,
  ajout personnel, édition, langue FR/EN, menu mobile et pied de page bilingue.
  Les données PRE sont la principale source de croissance : rester sobre à l'ajout.

• netlify.toml — config de déploiement. Commande : node build-bi.js. Dossier
  publié : site.

• REPRISE-lumen.txt — CE FICHIER. Tenu à jour à chaque session et livré avec
  les autres.

POINT D'ATTENTION (cause d'un bug récent) : le pied de page, la page « à propos »,
le comportement JS de la bibliothèque (ouverture des domaines/sous-sections) ET la
coupe des résumés trop longs dans les cartes de la bibliothèque (« voir plus »)
sont définis À LA FOIS dans index__48_.html / data-en.js ET dans build-bi.js.
Si build-bi.js prend du retard sur les deux autres, le site généré diverge du SPA.
Toujours vérifier la cohérence FR (build-bi.js) ↔ FR (index__48_.html) ↔ EN (data-en.js).

3. CONVENTIONS DE CONTENU & STYLE (éditoriales)
────────────────────────────────────────────────────────────────────
• Notifications (« Nouveautés », la cloche du menu) : elles annoncent SEULEMENT les
  VRAIES nouveautés, c'est-à-dire le contenu réellement nouveau qu'un lecteur peut aller
  lire ou utiliser : nouveaux articles, nouveaux outils ou fonctionnalités (ex. Mémoriser).
  PAS le superflu : ni approfondissement, refonte, renforcement, fusion, réorganisation,
  renvoi ou correction d'un contenu existant. Source : NOUVEAUTES dans index__48_.html
  (FR) et NOUVEAUTES_EN dans data-en.js (EN), groupes parallèles (même ordre, mêmes
  éléments traduits). Ajouter les vraies nouveautés en tête du groupe de date courant,
  dans les deux langues ; la cloche signale alors le changement aux visiteurs.
• Apostrophes : DROITES dans index__48_.html ; COURBES (’) dans build-bi.js et
  memoriser.html (chaînes JS).
• Références bibliques : forme standard, modèle « Genèse 44:2-3 ». Verset en italique
  entre guillemets, référence en gras (dans les articles).
• Mots grecs/hébreux : nommer la langue, le mot français/anglais traduit entre
  guillemets, translittération en italique, puis l'alphabet original si utile.
• Style : sobre, dense, catholique, accessible, sans ton scolaire artificiel.

────────────────────────────────────────────────────────────────────
3 ter. MANDAT PERMANENT — AUDIT CONTENU / MÉMORISER
────────────────────────────────────────────────────────────────────
À chaque session de modification de contenu, vérifier si les articles nouveaux ou
modifiés appellent une mise à jour de Mémoriser. Ne pas ajouter mécaniquement tous
les versets cités : privilégier les versets qui portent réellement la doctrine ou
la mémoire du sujet, éviter les doublons et les références purement contextuelles.
